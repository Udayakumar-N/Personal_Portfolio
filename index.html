🛡️ Women Safety System
Single-file Web App · Built from empathy
🔴 Live Working Prototype Try all features below ↓
⚠️ Working Prototype — All buttons are functional in this demo. SOS, location fetch, fake call, siren (audio), contact management, and activity log all work. Real SMS/call features require a mobile device or backend integration.
🚀 Features that can be added in future:
📱 Real SMS via Twilio API  ·  🗺️ Live GPS tracking link  ·  🔐 PIN-based secret trigger  ·  🤝 Community alert zone  ·  📸 Auto photo capture on SOS  ·  🧠 AI-based threat detection  ·  ⌚ Wearable device integration  ·  🌐 PWA (installable offline app)
❓ Why I Built This

Safety is a real concern for women, especially when travelling alone or in emergencies. I wanted to build something that didn't need an app download, didn't require login, and worked instantly. A single HTML file felt like the most accessible solution — anyone can open it from their phone browser.

🚧 Problems I Faced

The biggest challenge was making the emergency trigger truly fast — one tap, no friction. I also struggled with making the UI work well on small mobile screens under stress. Getting browser-based location access to work without a backend was tricky, and I had to find workarounds to avoid needing server infrastructure.

📚 What I Learned

I learned how to use the browser Geolocation API, how to trigger tel: and sms: links for emergency calls/messages, and how to design for stress — where simplicity literally matters. I also learned that a single well-built HTML file can be more powerful than a full-stack app for certain use cases.

Geolocation API Mobile UX Design Stress-first UI AudioContext API Single-file Architecture
✅ To-Do List App
Mini Project · My first real JS application
💡 Simple on the surface, but packed with learning. This was one of the first projects where I truly understood how JavaScript interacts with the browser DOM in real time.
❓ Why I Built This

Every developer's journey includes a To-Do app — and mine was no different. I built it not just as an exercise, but because I genuinely wanted to understand how to build something interactive from scratch. I also needed a simple task tracker I could actually use daily.

🚧 Problems I Faced

The hardest part was making tasks persist across page refreshes using localStorage. I also ran into issues with dynamically adding and deleting DOM elements — understanding event delegation was a real challenge at first. Making the UI feel responsive and not clunky took several iterations.

📚 What I Learned

This project taught me DOM manipulation fundamentals — createElement, appendChild, removeChild — and how to use localStorage for data persistence without a database. I also learned how to handle user input validation cleanly and write cleaner, event-driven JavaScript.

DOM Manipulation localStorage Event Listeners Input Validation JavaScript
🎮 Simple Browser Games
Mini Projects · Logic + Fun combined
💡 Games are the best way to practice logic — every mechanic is a small algorithm. Building these helped me understand loops, conditionals, timers, and state management in a hands-on way.
❓ Why I Built These

I wanted to apply programming logic in a fun, visual way. Games force you to think about state — what's happening right now, what changes when the user acts, and how to keep score. These small projects helped me bridge the gap between theory and real interactivity.

🕹️ Games Built

Built simple browser games including a Number Guessing Game (random logic, attempt tracking), a Rock Paper Scissors game (conditional logic, scoreboard), and a basic Quiz Game (arrays, dynamic rendering, timer). Each one taught a different concept.

Number Guessing Rock Paper Scissors Quiz Game
🚧 Problems I Faced

Managing game state was harder than expected — knowing when a game is "over", resetting correctly, and preventing bugs when users click rapidly. I also struggled with timers using setInterval and making sure they cleared properly to avoid memory issues.

📚 What I Learned

These games taught me how to manage application state in vanilla JavaScript, use Math.random() for randomness, work with setInterval/clearInterval for timers, and structure code so it's easy to reset and replay. Most importantly, I learned that even simple programs need careful state management.

State Management setInterval / Timers Math.random() Conditional Logic JavaScript
